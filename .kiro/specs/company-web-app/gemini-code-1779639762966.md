# Technical Specification & Development Plan

## 1. Project Overview
**Product Name:** Company Web Application (MVP Phase)
**Objective:** Rapid deployment of the frontend architecture for the multi-pillar tech venture.
**Target Deployment Time:** < 50 minutes.
**Hosting Environment:** GitHub Pages (Static Hosting).

## 2. System Architecture
As this phase relies on rapid iteration and static hosting, the architecture is strictly decoupled. The client-side application will operate independently, consuming external REST/GraphQL APIs for any dynamic data, authentication, or business logic as the product scales.

### Core Stack
* **Build Tool:** Vite (selected for ultra-fast Hot Module Replacement and optimized production builds)
* **Frontend Framework:** React.js
* **Routing:** React Router (Note: Use `HashRouter` to prevent 404 errors on direct navigation within GitHub Pages, or implement the 404.html redirection hack)
* **Styling:** CSS Modules or Tailwind CSS (recommended for rapid UI iteration)
* **Deployment:** `gh-pages` npm package automating pushes from the `dist` folder.

## 3. Directory Structure & Scalability
To maintain clean code standards suitable for scaling an enterprise application, the `src` directory is structured to separate concerns from day one:

    ├── public/             # Static assets (favicon, static JSON data, etc.)
    ├── src/
    │   ├── assets/         # Images, icons, and global stylesheets
    │   ├── components/     # Reusable, stateless UI components (Buttons, Cards, Nav)
    │   ├── features/       # Feature-specific modules isolating domain logic
    │   ├── hooks/          # Custom React hooks
    │   ├── layouts/        # Page layouts (e.g., MainLayout, DashboardLayout)
    │   ├── pages/          # Route-level components
    │   ├── services/       # External API integrations and fetch wrappers
    │   ├── store/          # Global state management (Context API / Zustand)
    │   ├── utils/          # Helper functions, constants, and formatters
    │   ├── App.jsx         # Root component and router configuration
    │   └── main.jsx        # Application entry point

## 4. Deployment Pipeline Configuration
Deployment is entirely Git-driven via the `gh-pages` branch. 

**Prerequisites in package.json:**
    "homepage": "https://<GITHUB_USERNAME>.github.io/<REPO_NAME>/",
    "scripts": {
      "predeploy": "npm run build",
      "deploy": "gh-pages -d dist"
    }

**Prerequisites in vite.config.js:**
    import { defineConfig } from 'vite'
    import react from '@vitejs/plugin-react'

    export default defineConfig({
      base: '/<REPO_NAME>/',
      plugins: [react()],
    })

## 5. Immediate Execution Milestones
1.  **Initialize Architecture:** Run Vite bootstrap and clean out boilerplate CSS/SVG files.
2.  **Configure Deployment:** Set up the `base` path and GitHub Pages scripts. Test deployment with a blank screen.
3.  **Establish Core Layout:** Build the overarching layout components to frame the application.
4.  **Integrate Mock State:** While backend APIs are pending, utilize static JSON payloads in the `/services` folder to populate the UI and test data structures.
5.  **Iterate UI:** Begin building out specific feature pages within the remaining time block.
